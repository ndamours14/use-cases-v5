**User Story:** As a new AVAnnotate user, I want to learn and understand the features of AVAnnotate so that I can build my own AVAnnotate projects.

**Description:** This Documentation Project is an excellent introduction to AVAnnotate’s features, explaining how and why AVAnnotate can be used. It demonstrates both how projects are constructed (Documentation) and how projects can be effectively presented to an audience (Narrative).

**Ingredients:**
1. The Monster Mash - Documentation Project
2. 1 or more audio or audiovisual recordings
3. Annotations
4. Tags

**Method:**
Read the Documentation and Narrative sections on each page of the Documentation Project in order to learn about AVAnnotate’s features
Using your own audio or audiovisual records, annotations, and tags, you can create your own AVAnnotate project using the information gleaned from the Documentation Project
Example: The Monster Mash - Documentation Project by AVAnnotate Team (this project was shared on the AVAnnotate GitHub discussion board by Tanya Clement - will seek permission from Tanya to share more widely if included on a public use case website)
